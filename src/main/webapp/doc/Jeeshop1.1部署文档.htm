Jeeshop1.1部署文档


jeeshop 发布于：2014-07-17 | 阅读4482次

一、所需环境

JDK1.6、tomcat6.0、mysql

二、项目部署

  1. 下载jeeshop源码。
    下载地址：http://git.oschina.net/jeeshop/jeeshop
  2. 建立jeeshop数据库和表结构
    找到下载的jeeshop目录jeeshop/web/doc下的sql文件git_jeeshop_20140702.sql在你本地的mysql工作空间执行，完成数据库和表结构的建立。
  3. 导入源码到eclipse工作空间，并且解决jar包的路径问题，然后build下。
  4. 修改src/applicationContext.xml下的数据库密码为你本地的mysql的数据库密码。
  5. 部署项目到tomcat
    比如我本地是E:\apache-tomcat-6.0.37\conf\server.xml， 则在server.xml文件的节点之间增加 
    <Context path="/jeeshop" docBase="D:\git\jeeshop\web" debug="0" reloadable="true" crossContext="true"/>
    docBase：为你jeeshop项目的路径 path：为你访问的项目名称。
  6. 启动tomcat，如果没有任何异常，则可以访问
    Tomcat启动完毕后，如果没有任何异常，则可以直接访问jeeshop的后台了。
    门户访问:http://ip:port/jeeshop

    后台访问:http://ip:port/jeeshop/manage/admin.jsp 

    后台账号
	admin/123456	管理员账号，拥有全部权限
	test1/123456	测试账号，只拥有查询权限
门户测试账号：test1/123456

二、配置图片服务器

jeeshop目前只支持使用阿里云的OSS作为图片服务器来存储图片，新手朋友去可以阿里云的官方网站http://www.aliyun.com/注册一个账号，然后开通存储服务OSS，如果是新账号阿里云一般是先免费给使用的。

开通OSS存储服务之后，需要手动创建jeeshop能够识别的目录结构。目录结构图如下：


目录结构编辑完成后，进行jeeshop的后台，在左侧菜单中找到【系统管理】--【系统设置】--图片设置，修改“图片根路径”为你申请的OSS存储服务的图片的跟路径。

比如你申请的OSS的创建的Bucket为jeeshop11，则你的图片跟路径为：http://jeeshop11.oss.aliyuncs.com ，填写好以后，点击保存，并重启服务或者手动加载缓存以使得修改生效。

最后，在jeeshop的后台【第三方插件】---【存储管理】中修改阿里云存储的配置信息为你自己的ID、KEY、bucketName。（查找自己OSS存储配置信息ID和KEY的方法为：登陆阿里云，进入控制台首页-----开放存储服务OSS，点击页面右上角的ACCESSKEY管理，然后创建自己的Access key。）


到此，你的jeeshop项目的图片服务器已经配置完毕。


四、配置jeeshop系统邮箱

邮箱对于任何一个系统都很重要，它负责发送注册、找回密码、修改密码、商品到货通知、商品降价通知以及等等等等。同样，邮箱对于jeeshop系统而言也很重要。

找到jeeshop/src/system.properties文件找到下面几行代码，就可以修改jeeshop所关联的邮箱信息了。 

from_email_account = xxx@163.com
from_email_password = xxx
from_eamil_smtpServer = smtp.163.com


五、常见问题

  1. 为什么本地启动系统，使用QQ或新浪微博登陆会出现空白页面。

    解答：QQ或新浪微博登陆必须在有公网IP的服务器上进行调试才行，因为这两个接口登陆成功会访问回调URL，找不到则会出现空白页面。

  2. 为什么本地启动系统后访问门户非常非常慢，甚至一分钟都不出来。

    解答：请确保本地能够上网，因为项目中使用到了网络字体库。

  3. git上下载的jeeshop有sql文件或者使用说明，数据库等设计文档之类的东西吗？

    解答：有的，在web/doc目录下面。

  4. 项目部署后会出现乱码？是否需要修改系统的什么配置？

    解答：不需要修改系统配置的，只需要修改启动容器的编码，比如tomcat是server.xml的Connector节点增加URIEncoding="UTF-8"属性。
